# 天津美术高考志愿辅助 — 技术方案

## 一、项目定位

**产品名称：** 天津美术志愿（TJArtVolunteer）
**包名：** com.tangtang.tjartvolunteer
**定位：** 服务天津市美术与设计类考生，**只推荐不需要校考的院校和专业**

## 二、核心功能

### 1. 智能推荐（核心）
- 输入：美术统考成绩、文化课成绩、选科、意向地域、兴趣标签
- 输出：冲/稳/保 三档推荐，只包含**认可天津统考成绩的院校**
- 算法：基于天津美术类历年录取综合分，结合用户偏好加权排序

### 2. 院校库（仅省统考院校）
- 在天津招收美术/设计类、**不组织校考**的院校
- 天津本地院校重点展示
- 院校详情：录取规则、往年综合分、招生计划

### 3. 专业查询
- 按学科门类筛选：美术学类、设计学类、戏剧与影视学类、艺术学理论类
- 专业详情：介绍、就业方向、薪资

### 4. 历年数据
- 天津美术类本科A/B批录取综合分（分专业）
- 数据可视化：折线图看趋势

### 5. 志愿表管理
- 创建多个志愿方案，拖拽排序，导出分享

### 6. 录取规则解读
- 各院校美术类录取规则（综合分计算公式）
- 天津市美术类投档规则

## 三、校考排除策略（重要！）

### 核心原则：按专业逐个排查，不按学校一刀切

**原因：** 同一所学校可能同时有校考专业和省统考专业，按学校排除会遗漏可报专业。

### 排除逻辑
```
对每个学校的每个专业，逐个判断：
  if (该专业在天津招生 且 该专业不要求校考) → 纳入推荐
  if (该专业在天津招生 且 该专业要求校考) → 排除
  if (该专业不在天津招生) → 排除
```

### 已确认数据（来自IMA知识库）

### ⚠️ 核心原则：只排除已确认的校考专业，其余全部纳入推荐

**绝不因不确定而排除任何专业。** 只有官方明确写"组织校考"的专业才排除。

## 四、技术架构

### 前端（Android）
- Kotlin + Jetpack Compose + Material3
- 图表：MPAndroidChart
- 导航：Jetpack Navigation

### 数据层
- **本地SQLite数据库**（预置天津美术类数据包）
- 首次启动从assets加载
- 支持在线更新（GitHub Release增量包）

### 推荐算法（纯Kotlin）
```
综合分 = f(美术统考成绩, 文化课成绩, 院校录取公式)
录取概率 = f(综合分差距, 历年波动)
推荐得分 = w1*录取概率 + w2*院校匹配 + w3*专业匹配 + w4*地域匹配
```

### 天津美术类综合分公式（重要！2025年已改革）
```
2025年（当前）：综合分 = 文化课成绩×50% + 专业课统考成绩×2.5×50%
2024年：综合分 = 文化课成绩×40% + 专业课统考成绩×2.5×60%
2023年：综合分 = 文化课成绩×40% + 专业课统考成绩×2.5×60%
```

### 校考排除逻辑（按专业粒度，宁多勿少）
```
对每个学校的每个专业，逐个判断：
  if (该专业已确认组织校考) → 排除（不显示）
  if (该专业已确认承认省统考) → 纳入推荐
  if (该专业在天津不招生) → 排除（不显示）
  if (校考状态未知) → 纳入推荐 + 标注"⚠️ 校考状态待确认"

核心原则：宁可多推荐（用户自行判断），不可漏掉可报专业
```

```
  admission_scores.json   # 天津历年录取数据
  employment.json         # 就业统计
  exclude_universities.json # 校考排除清单
  tianjin_universities.json # 天津本地院校详情
```

### 数据量估算
- 认可院校：~100所 × 2KB = 0.2MB
- 排除院校：~30所 × 2KB = 0.06MB
- 专业目录：~50个 × 1KB = 0.05MB
- 录取数据：~1万条 × 200B = 2MB
- 就业数据：~1000条 × 1KB = 1MB
- **总计约3.3MB**
