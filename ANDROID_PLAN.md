# NewProjectV2402 → 安卓APK 技术方案

## 一、原项目功能分析

### 核心功能模块
1. **日线预测** — 基于历史数据预测次日价格区间（含策略库加载、置信度）
2. **在线预测** — 实时监控+自适应预测（OnlinePredictor）
3. **日线回测** — 历史数据回测+图表生成（matplotlib）
4. **工具箱** — 资金管理、持仓管理、日线数据下载、状态管理、算法诊断
5. **股票切换** — 从持仓列表选择或手动输入代码，自动下载数据
6. **数据源** — Tushare API + 新浪/网易/搜狐实时行情
7. **策略引擎** — 12种统计策略（KDJ、MACD、RSI、布林带、ATR通道等）
8. **LMSR市场做市** — 流动性市场评分规则
9. **周易预测** — 易经预测模块
10. **回测系统** — 策略引擎、绩效分析、参数优化、在线学习

### 技术栈
- Python 3.13, PyQt5（实际是CLI）
- pandas, numpy, matplotlib, tushare
- 数据本地CSV存储

## 二、安卓APK技术方案

### 架构：Kotlin原生 + Python后端API

```
┌─────────────────────────────────────────┐
│           Android App (Kotlin)           │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 首页    │ │ 预测页   │ │ 回测页   │ │
│  │ 股票列表│ │ 价格图表 │ │ 绩效报告 │ │
│  └─────────┘ └──────────┘ └──────────┘ │
│  ┌─────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 工具箱  │ │ 设置页   │ │ 持仓管理 │ │
│  └─────────┘ └──────────┘ └──────────┘ │
└──────────────────┬──────────────────────┘
                   │ HTTP/REST API
┌──────────────────┴──────────────────────┐
│         Python FastAPI 后端              │
│  ┌──────────┐ ┌────────┐ ┌───────────┐ │
│  │ 策略引擎 │ │ 数据层 │ │ 回测引擎  │ │
│  │ (移植)   │ │(tushare│ │ (移植)    │ │
│  └──────────┘ └────────┘ └───────────┘ │
└─────────────────────────────────────────┘
```

### 方案A：完整原生（推荐，工作量大）
- Android端：Kotlin + Jetpack Compose + MPAndroidChart + Retrofit
- 后端：FastAPI 服务（部署到云服务器）
- 数据：tushare API + 实时行情API
- 构建：GitHub Actions → APK

### 方案B：混合方案（折中）
- 用 Chaquopy（Python SDK for Android）直接在安卓跑Python
- 不需要后端服务器
- 但包体积大，性能一般

### 选定方案：方案A（原生 + API后端）

## 三、任务拆分

### Phase 1: 后端API (Worker 1)
- 将Python策略引擎封装为FastAPI REST API
- 端口：预测、回测、数据获取、持仓管理
- 部署到Render/Railway免费托管

### Phase 2: Android UI框架 (Worker 2)
- Kotlin + Jetpack Compose 项目搭建
- 首页、导航、股票列表
- 网络层（Retrofit + OkHttp）

### Phase 3: 核心功能页 (Worker 3)
- 预测页（价格图表 + 预测结果）
- 回测页（绩效报告 + 图表）
- 持仓管理页

### Phase 4: 工具箱 + 设置 (Worker 4)
- 工具箱页面
- 设置页（Tushare token配置）
- 数据下载功能

### Phase 5: 整合 + 构建 (Worker 5)
- GitHub Actions CI/CD
- 自动构建APK
- 签名 + 发布
