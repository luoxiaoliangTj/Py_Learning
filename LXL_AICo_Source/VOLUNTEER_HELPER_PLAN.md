# 高考志愿辅助App — 技术方案

## 一、项目定位

**产品名称：** 志愿助手（VolunteerHelper）
**包名：** com.tangtang.volunteer
**定位：** 高考志愿填报决策辅助工具，综合分数、兴趣、地域、就业前景，给出个性化推荐

## 二、核心功能

### 1. 智能推荐（核心）
- 输入：分数、位次、选科、意向地域、意向专业、兴趣标签
- 输出：冲/稳/保 三档推荐，每档包含学校+专业+录取概率
- 算法：基于历年录取位次进行概率估算，结合用户偏好加权排序

### 2. 院校查询
- 按省份/类型/层次筛选（985/211/双一流/普通本科/专科）
- 院校详情：基本信息、王牌专业、往年录取线、院校排名

### 3. 专业查询
- 按学科门类/专业类筛选
- 专业详情：专业介绍、就业方向、薪资水平、考研方向

### 4. 历年数据
- 分省分批次分科目的录取分数/位次
- 数据可视化：折线图看趋势

### 5. 志愿表管理
- 创建多个志愿表（如"第一套方案""备选方案"）
- 拖拽排序，导出分享

### 6. 就业数据
- 各专业就业率、平均薪资、热门行业
- 数据来源：教育部就业报告、第三方统计

## 三、技术架构

### 前端（Android）
- Kotlin + Jetpack Compose + Material3
- 图表：MPAndroidChart
- 导航：Jetpack Navigation

### 数据层
- **本地SQLite数据库**（预置数据包）
- 首次启动时从assets加载预置数据
- 支持在线更新数据包（从GitHub Release下载增量包）

### 推荐算法（纯Kotlin实现）
```
录取概率 = f(位次差距, 历年波动, 招生计划变化)
综合得分 = w1*录取概率 + w2*院校匹配 + w3*专业匹配 + w4*地域匹配 + w5*就业匹配
```
- 位次差距：用户位次 vs 历年录取位次中位数
- 历年波动：标准差衡量稳定性
- 权重可用户自定义

### 数据获取策略
- **预置数据包**：从教育部阳光高考公开数据整理，打包在assets
- **在线更新**：GitHub Release发布增量数据包
- **不依赖后端**：和LXL.AICo一样，完全离线运行

## 四、数据方案

### 数据来源
1. 教育部阳光高考平台（院校信息、专业目录）— 公开
2. 各省考试院历年录取数据 — 公开，需整理
3. 教育部就业质量报告 — 公开
4. 院校官网（招生计划、章程）— 公开

### 数据格式
```
assets/data/
  provinces.json          # 省份列表
  universities.json       # 院校信息（3000+所）
  majors.json             # 专业目录（800+个）
  admission_scores.json   # 历年录取数据（按省份分文件）
  employment.json         # 就业统计数据
```

### 数据量估算
- 院校信息：~3000所 × 2KB = 6MB
- 专业目录：~800个 × 1KB = 0.8MB
- 录取数据：~50万条 × 200B = 100MB（按省份分片，单省约2MB）
- 就业数据：~5000条 × 1KB = 5MB
- **总计约110MB**，首次下载后完全离线

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 数据准确性 | 标注数据来源，提供反馈入口 |
| 数据时效性 | 每年高考后更新数据包 |
| 推荐算法不准 | 算法透明可解释，用户可手动调整 |
| 数据量大 | 按省份分片，按需加载 |
